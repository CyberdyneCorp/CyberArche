# Design Brief: CyberArche — AI-Centric Collaborative Document Workspace

> Prompt to hand to a design assistant (e.g. Claude) to produce the visual
> design system and key screens for the CyberArche frontend. Grounded in the
> `bootstrap-cyberarche` OpenSpec change (see `openspec/changes/bootstrap-cyberarche/`).

## Role

You are a senior product designer. Design a beautiful, modern, cohesive UI for a
web app called **CyberArche**. Produce a full visual design system and the key
screens described below. Optimize for clarity, focus, and a premium, technical
feel — this is a power tool for technical teams, not a toy.

## Product in one line

CyberArche is an AI-centric, real-time collaborative document platform — a
Notion/Confluence alternative where math, diagrams, whiteboards, code, and an AI
agent are first-class, and multiple people (and the AI) edit the same document
live.

## Target users

Engineering, research, and product teams who write technical docs: RFCs, specs,
runbooks, research notes, design docs. They value keyboard-driven speed, dense
but legible information, and trustworthy AI assistance.

## Brand & tone

- Modern, precise, "engineering-grade." Calm and focused, not flashy.
- Confident use of whitespace; content is the hero, chrome recedes.
- A restrained accent color for interactive/AI moments; neutral grays for surface.
- Feels fast: subtle micro-interactions, no heavy skeuomorphism.
- Suggest a color palette (light + dark themes are both REQUIRED and must be
  designed as equals), a type scale (a clean sans for UI, and a good monospace
  for code/LaTeX/data), spacing scale, radius, elevation, and iconography style.

## Core layout (the app shell)

A three-region workspace:

1. **Left sidebar** — collapsible. Workspace switcher at top; a nested,
   drag-reorderable document tree (documents can contain documents); sections for
   Favorites, Shared with me, and Trash; a global "＋ New" and a search entry.
2. **Center — the document canvas** — the block editor (the main event). Clean
   reading width, generous margins, a slim breadcrumb + title area on top, and a
   presence stack (live collaborator avatars) top-right.
3. **Right panel — the AI Agent** — dockable/collapsible. Chat + actions with the
   document's agent. Can be hidden to give the editor full width.

Design the collapsed and expanded states of both side panels, and a focused
"just the document" mode.

## The block editor (design this in depth — it's the centerpiece)

A block-based rich-text editor (think Notion) where every paragraph, heading, or
media is a "block." Design:

- **The slash (`/`) command menu**: a fast, searchable popover to insert any
  block type, grouped and with icons + short descriptions.
- **Block hover controls**: a drag handle and a "＋" add button that appear on
  hover at the left gutter; a right-click / "⋮⋮" context menu (duplicate, delete,
  turn into, comment, move).
- **Block selection & drag-and-drop** reordering with a clear drop indicator.
- **Inline formatting toolbar** (bubble menu) on text selection: bold, italic,
  code, link, color, and "Ask AI" on the selection.
- **Empty state** for a new document (placeholder + hint to type "/" or "@").

### Block types to design (show each rendered + its editing affordance)

- **Text**: paragraph, H1–H3, bulleted/numbered lists, to-do checkboxes, quote,
  callout (with an icon + colored background), divider.
- **Code block**: language selector, syntax highlighting, copy button, line
  numbers. Use the monospace type.
- **LaTeX / math**: both inline math and a centered block equation, beautifully
  typeset; show the raw-source editing state and an inline error state.
- **Mermaid diagram**: rendered diagram (flowchart) with a toggle between
  rendered view and editable source; show a parse-error state.
- **Table**: clean data table with add/remove row & column controls, resizable
  columns, and rich-text cells. Also show a table that was auto-generated from an
  imported CSV/Excel sheet.
- **Whiteboard block** (see below).
- **Image / file / embed** blocks.
- **AI block**: a block whose content was generated by the agent, subtly marked
  as AI-authored with provenance.

## Whiteboard / mind-map (an embedded canvas block)

An Excalidraw-style infinite canvas embedded inside a document. Design:

- The canvas toolbar (select, shapes: rectangle/ellipse/diamond, arrow/connector,
  freehand pen, text, image), color/stroke/fill controls, zoom controls.
- A hand-drawn-but-tidy aesthetic for shapes; connectors that stay bound to shapes.
- A **mind map** example: a root node with branches, plus an "AI: generate mind
  map from this document" action.
- Fullscreen/expanded canvas mode vs inline-in-document mode.

## Real-time collaboration (make presence feel alive)

- **Live cursors** of other people with name labels and per-user colors.
- **Text selections** highlighted in collaborators' colors.
- **Presence avatar stack** with online status; overflow "+N".
- A subtle **connection status** indicator (online / reconnecting / offline —
  "editing offline, will sync").
- **Comments**: a comment thread anchored to a block, a comment marker in the
  gutter, resolve/reopen, and a comments overview panel.
- Show the AI agent appearing as a collaborator (its own colored cursor/avatar)
  while it edits live.

## The AI Agent panel (a key differentiator — design richly)

A right-side panel where the user works with the document's AI agent:

- **Conversation** thread (user + agent messages), with the agent's answers
  citing the blocks/sources it used (citation chips).
- **Quick actions**: Summarize, Draft, Rewrite selection, "Create mind map,"
  "Ingest a file."
- **Insert affordances**: agent output can be previewed and inserted into the
  document as blocks (show an "Insert" / "Replace selection" control).
- **File ingestion**: a drag-and-drop / upload zone for PDF, CSV, Excel; show an
  ingestion progress state (uploading → processing → done) and the resulting
  content/table appearing in the doc.
- **Grounding indicator**: a subtle marker that the agent is grounded in the
  workspace knowledge (RAG), plus a way to scope context (this doc vs workspace).
- **Model selector** (provider-agnostic — a small, unobtrusive dropdown).
- **Tools / MCP**: an area showing which tools the agent can use, including
  user-attached external MCP servers (see below), with per-session enable toggles.
- **Run history / audit**: a way to review past agent runs (prompt, tools used,
  what it changed).

## MCP connectors (settings surface)

A settings screen to **attach external MCP servers** so the agent gains extra
tools: a list of connectors (name, origin, status), an "Add connector" form
(endpoint + credentials, with credentials shown as secured/masked), per-connector
enable/disable, and namespaced tool listings.

## Sharing & permissions

- A **Share dialog**: invite people by identity, assign roles (Owner, Editor,
  Commenter, Viewer), manage a shareable link with a permission level
  (view/comment/edit), link expiry, and revoke.
- Role/permission badges shown contextually.

## Auth / entry

- A clean **sign-in screen** (SSO via an external identity provider — single
  "Continue with SSO" primary action), and a first-run/empty workspace state.

## Cross-cutting requirements

- **Light and dark themes**, both fully designed and equally polished; define the
  theme tokens.
- **Responsive**: primary target is desktop, but show how the sidebar/agent panel
  collapse for narrower widths and a tablet layout.
- **Accessibility**: sufficient contrast (AA+), visible focus states, and keyboard
  affordances (this is a keyboard-driven app — show shortcut hints where relevant,
  e.g. a command palette).
- **States**: design empty, loading, error, and offline states for the key
  surfaces (document, agent panel, ingestion, connection).
- **Command palette** (⌘K): global search + quick actions.
- **Micro-interactions**: describe the motion for slash menu, block drag,
  presence, agent streaming, and ingestion progress — subtle and fast.

## Deliverables I want from you

1. A **design system / style guide**: color palette (light + dark), typography,
   spacing, radii, elevation, iconography, and reusable components (buttons,
   inputs, menus, popovers, toasts, avatars, badges, tabs, toggles).
2. High-fidelity mockups of these screens/states:
   - App shell (sidebar + document + agent panel), light and dark.
   - The block editor showing a rich document that uses text, a callout, a code
     block, a LaTeX equation, a Mermaid diagram, a table, and an embedded
     whiteboard/mind map.
   - The slash command menu and the inline selection toolbar.
   - The whiteboard/mind-map canvas (inline + fullscreen).
   - Live collaboration: cursors, selections, presence stack, a comment thread.
   - The AI Agent panel: conversation with citations, quick actions, file
     ingestion in progress, and inserting output as blocks.
   - Share dialog and MCP connectors settings.
   - Sign-in and empty-workspace first-run.
3. A short rationale for the key visual and interaction decisions.

Prioritize a cohesive, premium feel where the document and the AI are the stars
and the chrome stays quiet. Start by proposing the design system, then the app
shell, then the editor.
